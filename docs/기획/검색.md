# 검색 ✅

## 5.1 개요

검색 기능은 사용자가 파일을 빠르게 찾을 수 있도록 지원하는 기능으로,

**현재 폴더 범위 내 검색**과 **전체 범위 검색**을 제공한다.

검색 기능은 파일 관리 모듈과 밀접하게 연동되며,

검색 결과를 통해 파일의 실제 위치로 이동할 수 있도록 설계한다.

검색은 **파일명 및 폴더명을 기준으로 수행되며**,

파일 내용, 메타데이터, 태그 기반 검색은 제공하지 않는다.

---

## 5.2 폴더 내 검색

### 5.2.1 기능 설명

- 현재 선택된 폴더 범위 내에서 파일 및 하위 폴더를 검색한다.
- 검색어 입력 시 파일 목록을 실시간으로 필터링한다.

### 5.2.2 동작 정책

- 검색은 **파일명 및 폴더명 기준**으로 수행한다.
- 부분 일치 기반 문자열 검색을 사용한다.
- 검색어가 적용된 상태에서는 전체 목록이 아닌 검색 결과만 표시된다.
- 검색어를 초기화하면 원래의 파일 목록으로 복귀한다.

---

## 5.3 전체 검색

### 5.3.1 기능 설명

- 전체 검색은 별도의 검색 모달을 통해 제공된다.
- 사용자는 프로젝트 및 폴더 구조 전체를 대상으로 파일을 검색할 수 있다.

### 5.3.2 검색 범위 및 기준

- 검색 대상:
    - 파일명
    - 폴더명
- 검색 제외 대상:
    - 파일 내용
    - 파일 메타데이터(설명, 태그 등)
    - 내부 시스템 정보

### 5.3.3 검색 결과 처리

- 검색 결과는 **파일 단위로 표시**된다.
- 검색 결과에서 파일을 선택하면 해당 파일이 포함된 폴더로 이동한다.
- 이동 시 폴더 컨텍스트가 갱신되며, 파일 목록이 다시 조회된다.

---

## 5.4 권한 및 접근 정책

- 검색 결과에는 사용자가 접근 권한을 보유한 파일 및 폴더만 포함된다.
- 접근 권한이 없는 리소스는 검색 결과에 노출되지 않는다.
- 검색 결과는 **권한 필터링 이후의 결과만 반환**하는 것을 원칙으로 한다.

---

## 5.5 성능 및 처리 원칙

- 검색 성능을 우선으로 고려하여 단순 문자열 검색 방식을 사용한다.
- 대소문자는 구분하지 않는다.
- 검색 결과는 기본 50개 단위로 제공되며, 추가 결과는 스크롤을 통해 로드된다.

---

## 5.6 예외 처리

- 검색 결과가 없는 경우, 검색 결과 없음(Empty State)을 표시한다.
- 검색어가 없는 경우 전체 목록을 표시한다.
- 네트워크 또는 시스템 오류 발생 시 오류 메시지를 표시하고 재시도를 안내한다.

---

### ✅ 5장 요약

- 검색 기능은 **파일명 및 폴더명 기준**으로만 동작한다.
- 현재 폴더 검색과 전체 검색을 모두 제공한다.
- 검색 결과 선택 시 파일의 실제 위치로 이동할 수 있다.
