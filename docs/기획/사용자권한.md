# 사용자 및 권한 관리 ✅

## 9.1 모듈 개요

사용자 및 권한 관리 모듈은 Portal NAS 시스템 전반에 적용되는

**사용자 계정 및 권한(Role)을 관리하는 공통 관리 모듈**이다.

본 모듈은 각 기능 모듈(파일 관리, 외부 공유, NAS 모니터링 등)이

권한 판단의 기준으로 삼는 **단일 권한 소스(Source of Truth)** 역할을 수행한다.

---

## 9.2 사용자 유형 정의

Portal NAS 시스템은 다음과 같은 사용자 유형을 정의한다.

| 사용자 유형 | 설명 |
| --- | --- |
| Admin | 시스템 전체 관리 권한을 가진 사용자 |
| Manager | 프로젝트 및 파일 관리 권한을 가진 사용자 |
| User | 파일 활용 중심의 일반 사용자 |

---

## 9.3 권한(Role) 계층 구조

사용자 권한은 계층 구조를 따른다.

```
Admin ⊃ Manager ⊃User

```

### 권한 계층 원칙

- 상위 권한은 하위 권한의 모든 기능을 포함한다.
- 각 기능 모듈은 이 권한 계층을 기준으로 접근 가능 여부를 판단한다.

---

## 9.4 사용자 관리 기능

### 9.4.1 사용자 목록 조회

- 시스템에 등록된 사용자 목록을 조회할 수 있다.

---

## 9.5 권한(Role) 관리 기능

### 9.5.1 사용자 권한 부여

- 사용자에게 다중 권한(Role)을 부여할 수 있다.
- 권한 변경 시 즉시 시스템 전반에 반영된다.

---

### 9.5.2 사용자 권한 변경

- 관리자는 사용자의 권한을 변경할 수 있다.
- 권한 변경 시, 변경된 권한 기준으로 즉시 기능 접근이 제한 또는 허용된다.

---

### 9.5.3 권한 변경 제한

- 시스템 기본 권한(Admin, Manager, User)은 고정된 권한으로 관리한다.

---

## 9.6 권한 변경 시 시스템 영향 범위

- 권한 변경은 기존 데이터에 영향을 주지 않는다.
- 권한 변경 이후 수행되는 모든 기능은 변경된 권한을 기준으로 판단한다.

---

## 9.7 보안 및 접근 제어 원칙

- 사용자 권한 관리 기능은 Admin 권한 사용자만 접근 가능하다.

---

## 9.8 타 모듈과의 관계

본 모듈은 다음 기능 모듈과 연계된다.

- **파일 관리 모듈**
    - 파일 이동, 삭제 등 권한 판단 기준 제공
- **외부 공유 모듈**
    - 공유 설정 가능 사용자 판단
- **NAS 모니터링**
    - 관리자 접근 제어 기준 제공
- **External Access**
    - 내부 사용자 권한과 분리된 정책 적용

각 모듈은 사용자 권한 판단 시 본 모듈에서 정의된 권한 체계를 참조한다.

---

### ✅ 10장 요약

- 사용자 및 권한 관리는 **시스템 공통 관리 모듈**이다.
- 모든 기능 모듈은 이 권한 체계를 기준으로 동작한다.
- 권한 변경은 즉시 반영된다.
- 외부 사용자는 본 모듈의 관리 대상이 아니다.
